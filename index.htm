RMS Support Tool
Microsoft© Customer Service & Support EMEA

The 'RMS Support Tool' provides the functionality to reset all Microsoft® AIP/AD RMS client services. Its main purpose is to delete the currently downloaded policies and reset the user settings for AIP/AD RMS service, and it can also be used to collect data for troubleshooting.

 

Introduction

This information gives you background and usage information for the RMS_Support_Tool Microsoft® Windows® PowerShell script module.

Note:

  • Please only run RMS_Support_Tool if you have been prompted to do so by a Microsoft® support technician.
  • It is recommended to test the RMS_Support_Tool with a test environment before executing it in a live environment.
  • Do not modify any component of the RMS_Support_Tool in any kind, as this will result in incorrect information in the analysis of your environment.
  • There is no support for the RMS_Support_Tool. Please see the disclaimer below.
  • Nomenclature:
    • RMS = Rights Management Service.
    • AD RMS = Active Directory Rights Management Service.
    • AIP = Azure Information Protection.
    • MSIP/MIP = Microsoft® Information Protection
    • MSIPC = Microsoft® Information Protection Client.
    • DRM = Digital Rights Management

Requirements

Before you can use the RMS_Support_Tool make sure that your environment fulfils the following requierements. Please update your environment if necessary.

Note:

  • The RMS_Support_Tool supports the following operating systems:
    • Microsoft® Windows® 8, Windows® 8.1, Windows® 10, Windows® Server 2012, Windows® Server 2012 R2, Windows® Server 2016 and Windows® Server 2019.
  • The RMS_Support_Tool supports the following Microsoft® Office versions:
    • Microsoft® Office 2010, Microsoft® Office 2013, Microsoft® Office 2016, Microsoft® Office 2019, and Microsoft® Office 365.

1. Microsoft® Azure Information Protection cmdlets (optional)

The Microsoft® Azure Information Protection cmdlets are installed with the Azure Information Protection client. Please ensure to have the latest version of the "AzureInformationProtection" cmdlets installed by checking its client version release history.

Currently, if you install the cmdlets as one user and run the cmdlets on the same computer as another user, you must first run the following command with administrative permissions in a Windows® PowerShell command window:

PS C:\> Import-Module AzureInformationProtection

2. Microsoft® Windows® 10 (recommended)

The RMS_Support_Tool offer functionality to reset all Microsoft® AIP/AD RMS services. While doing so, possible custom configurations will be backed up for you. The corresponding backup file (ServiceLocationBackup.reg) or its content can exceed the default path length of 260 characters and with that, backup or restore of custom configurations will fail. Other files may also be affected.

To prevent you from running into potential issues, it’s recommended to use the RMS_Support_Tool on a Microsoft® Windows® 10, version 1607 (or higher), or Windows® Server 2016 (or higher) operating system.

IMPORTANT: If you are using a different Microsoft® Windows® operating system, you must back up the following registry key manually:

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\MSIPC\ServiceLocation]

To use cmdlets with path lengths greater than 260 characters, run the Windows® Policy editor (gpedit.msc) and set the following group policy setting that is available only on the given operating systems:

Local Computer Policy > Computer Configuration > Administrative Templates > System > FileSystem > Enable Win32 long paths

For Windows® Server 2016 (and 2019), you must first install the latest Administrative Templates (.admx) for Windows® 10 before you can apply this policy.

For more information, see the Maximum Path Length Limitation section from the Windows® 10 developer documentation.

3. Internet access (required)

The RMS_Support_Tool consumes the following sources from the internet:

For logging and tracing purposes, the tool DebugView from Windows® Sysinternals is required. If you call the help file, the RMS_Support_Tool downloads its help file (RMS_Support_Tool.htm), and to check for updates, it downloads a file as well (Update.dat).

If you do not have the corresponding files on your local hard drive, the RMS_Support_Tool will automatically try to download the following files from the internet:

  • DebugView v4.81
  • RMS_Support_Tool.htm
  • Update.dat

Note:

  • If you’re using the RMS_Support_Tool in an environment that does not have internet access, you must download the files manually, before proceeding the RMS_Support_Tool.
  • You must place the files to the location where you have stored the RMS_Support_Tool files, and DebugView must be uncompressed into the subfolder “DebugView”.

 

Installation via PowerShell Gallery (recommended)

RMS_Support_Tool is available on PowerShell Gallery.

To be able to install packages from PowerShell Gallery you need to have the latest version of the PowerShellGet-module installed:

  • Installing PowerShellGet.

You must run the following command with administrative permissions, either in a Windows® PowerShell command window or in Windows® PowerShell ISE, to install the RMS_Support_Tool:

PS C:\> Install-Module -Name RMS_Support_Tool

 

Manual Installation

To install the RMS_Support_Tool manually, you must copy all its files (RMS_Support_Tool.psm1, RMS_Support_Tool.psd1, and RMS_Support_Tool.htm) to the following directory:

%ProgramFiles%\WindowsPowerShell\Modules\RMS_Support_Tool

Note:

  • This path must be listed in the PSModulePath environment variable. For more information, please read how to view the PSModulePath variable.

Sysinternals DebugView

IMPORTANT: If you're working in an environment that does not have internet access, in addition you must copy all files of the DebugView tool from Windows® Sysinternals (Dbgview.exe, dbgview.chm, and Eula.txt) into the following directory:

%ProgramFiles%\WindowsPowerShell\Modules\RMS_Support_Tool\DebugView

Check installation

To check if the installation was successful, you can call the Get-Module cmdlet with the following parameter:

PS C:\> Get-Module -ListAvailable

If you find an entry like the following, the installation was successful:

PS C:\> Get-Module -ListAvailable

 

    Directory: C:\Program Files\WindowsPowerShell\Modules

 

ModuleType Version   Name                               ExportedCommands

---------- -------   ----                               ----------------

Script     1.1.1     RMS_Support_Tool                   RMS_Support_Tool

 

Usage

You must run the RMS_Support_Tool with administrative permissions, either in a Windows® PowerShell command window or in Windows® PowerShell ISE. To do this, click the Microsoft® Windows® START button and start typing "power". While you type, you'll see suggestion "Windows PowerShell" (Desktop app). Right-click this entry. In the context menu, select "Run as administrator".

By default, running PowerShell scripts is disabled, and you must bypass this restriction in order to be able to run the RMS_Support_Tool. To do this, run the "Set-ExecutionPolicy"-command with the following parameters:

PS C:\> Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned -Force

Know issues:

  • Set-ExecutionPolicy has no effect. If you downloaded the RMS_Support_Tool from an untrusted website, it will be extended by an Alternate Data Streams in NTFS (ADS). If the corresponding website in the ADS is not trusted in your environment, the PowerShell command "Set-ExecutionPolicy" has no effect. There're two methods to solve this issue: Either you right-click the downloaded script file and "Unblock" it, or you add the correcponding website to your trusted sites settings in Internet Explorer.
  • Script does not start with the error: Set-ExecutionPolicy : PowerShell updated your local preference successfully, but the setting is overridden by the group policy applied to your system or Set-ExecutionPolicy : Windows PowerShell updated your execution policy successfully, but the setting is overridden by a policy defined at a more specific scope.. If you see this message in a PowerShell command window, you are most likely affected by the Windows® Group Policy setting "Turn on Script Execution". To resolve this problem, you must remove this Group Policy setting or configure a setting that allows scripts to run. Please request assistance from your administrator to do this. Please also read Use Group Policy to Manage Execution Policy and Set-ExecutionPolicy.

To start the RMS_Support_Tool, simply type "RMS_Support_Tool" into the PowerShell command window, and press enter. For example:

PS C:\> RMS_Support_Tool

When you start the RMS_Support_Tool, you'll see the following default menu:

  [1] INFORMATION

  [2] DISCLAIMER

  [3] HELP

  [4] RESET

  [5] USER LOGGING

  [6] ADMIN LOGGING

  [7] COMPRESS LOGS

  [8] EXIT MENU

 

Please select an option and press enter:

You can also start the RMS_Support_Tool within the command-line. Use the following command-line parameter to see a short summary of all available command-line parameters:

PS C:\> RMS_Support_Tool -Information

To see a full list of all command-line parameters with description, run the following command:

Get-Help RMS_Support_Tool -Detailed

To see the help for a single command-line parameter, for example for "-CompressLogs", run the following command:

Get-Help RMS_Support_Tool -Parameter CompressLogs

 

User experience

RMS_Support_Tool provides the following features (menu entry / command-line parameter):

[1] INFORMATION / -Information

This shows syntax and a description.

[2] DISCLAIMER / -Disclaimer

This displays the disclaimer of warranty.
Please read it carefully, and act accordingly.

[3] HELP / -Help

This opens the help file (RMS_Support_Tool.htm).

[4] RESET / -Reset

IMPORTANT: Before you proceed with this option, close all applications that run or use Microsoft® Office or AIP/AD RMS.

This option removes all AIP/AD RMS certificates, policy templates and all corresponding registry keys. Before, the RMS_Support_Tool creates a backup copy of existing custom configurations from the following registry key:

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\MSIPC\ServiceLocation]

The name of the backup file is “ServiceLocationBackup.reg”.

IMPORTANT: Please review point 2 in the prerequisites section of this help file for additional information.

Note:

  • When an Office 2013 installation is detected, modern authentication (ADAL) is automatically enabled as a precaution.

- / -SilentReset

This command line-parameter does the same as "-Reset", but does not print any output - unless an error occurs when attempting to reset.

Note:

  • If a silent reset triggers an error, you can use the additional parameter "-Verbose" to find out more about the cause of the error:

    PS C:\> RMS_Support_Tool -SilentReset -Verbose

[5] USER LOGGING / -UserLogging

IMPORTANT: Close all applications that run or use Microsoft® Office or AIP/AD RMS before you proceed with this option.

In a first step, this parameter activates the required logging, tracing or debugging mechanisms by implementing registry settings, and enabling some Windows event logs. This process will be reflected by a progress bar “Enable logging...".

In a second step asks you to reproduce the problem. While you’re doing so, the RMS_Support_Tool collects and records data. Once you have reproduced the problem, all collected files will be stored into the default logs folder (%temp%\RMS_Support_Tool\Logs). Every time you call this option, a new unique subfolder will be created in the logs-folder that reflects the date and time when it was created, e.g. “190221-133005”. While the files are being cached, you will see a progress bar “Collecting logs...".

In the last step, the RMS_Support_Tool resets all activated log, trace, and debug settings to their defaults. This process will be reflected by a progress bar “Disable logging...".

You can then review the log files in the logs folder.

Note:

  • An additional tool from Windows® Sysinternals (DebugView) is required for logging.
  • If you do not have this tool on your local hard drive, it will be automatically downloaded from the internet.

[6] ADMIN LOGGING / -AdminLogging

This parameter collects only AIP service configuration information of your tenant.
Please request assistance from your administrator to proceed this option.

Note:

  • You need to know your Microsoft® Office 365 global administrator account information to proceed with this option, as you will be asked for your credentials.
  • An additional tool from Windows® Sysinternals (DebugView) is required for logging.
  • If you do not have this tool on your local hard drive, it will be automatically downloaded from the internet.

[7] COMPRESS LOGS / -CompressLogs True

IMPORTANT: This command-line parameter should always be used at the very end of a scenario. Otherwise the compressed .zip file will not be created.

This parameter compress all collected logs files into a .zip file, and the corresponding path and file name is displayed.
In addition, the default logs folder (%temp%\RMS_Support_Tool\Logs) will be cleaned.
After this step you can review the log files, and send the collected logs to Microsoft® support technician.

[8] EXIT / -

This option will asks you whether you want to exit the menu after a confirmation prompt.

Note:

  • This option is not available in the command-line.

- / -Menu

This will start the RMS_Support_Tool with the default menu.

Note:

  • This option is not available in the menu.

- / -Version

This parameter displays the version of the RMS_Support_Tool, and offers a hyperlink to check for the latest version of the RMS_Support_Tool.

Note:

  • This option is not available in the menu.

 

Script log file

The RMS_Support_Tool creates the following log file. The log file give an overview about the commands that were run and offers a control mechanism to review the results.

Script.log

This log file collects the actions that has been taken and lists the results. It also logs general environment information from the used client machine, like the Windows® edition or Windows® PowerShell version.

 

Recorded log files

The RMS_Support_Tool collects the following log files and folders. If you want to learn more about the method used to collect a log or folder, please review the source code of the RMS_Support_Tool script module and read the comments.

Note:

  • Please note that not all log files are collected or recorded at all times. It depends on your environment which logs, or traces can be recorded.

AipServiceConfiguration.log

This file is only collected if you selected [6] ADMIN LOGGING. The file contains the Azure Information Protection service configuration of your tenant. For more information, please see the following documentation: Get-AipServiceConfiguration.

Tasklist.log

This file contains a list of currently running processes on the local computer. For more information, please see the following documentation: tasklist.

WinIPConfig.txt and IPConfigAll.log

These files show all current TCP/IP network configuration settings. For more information, please see the following documentation: ipconfig.

NetMon.etl

This file is a network trace recorded by Netsh.exe. For more information, please see the following documentation: Netsh Command Syntax, Contexts, and Formatting.

ProblemSteps.zip

This file records the exact steps you took when the problem occured. It was created by the Steps Recorder (psr.exe). For more information, please see the following documentation: Record steps to reproduce a problem.

MSIPC-MSDRM-DebugView.log

This file contains debug output for all MSIPC- or MSDRM-clients on your local system. It was created by DebugView.exe. For more information, please see the following documentation: DebugView v4.90.

MSIP/MSIPC/DRM logs

The following log folders contain more MSIP/MSIPC/DRM logging information:

%LOCALAPPDATA%\Microsoft\MSIP

%LOCALAPPDATA%\Microsoft\MSIPC

%LOCALAPPDATA%\Microsoft\DRM

For more information, please see the following documentation: How-to: debug a rights-enabled application.

AIP plugin for Adobe Acrobat logs

The following log folders contain logging information:

%LOCALAPPDATA%\Microsoft\RMSLocalStorage\MIP\logs

%USERPROFILE%\appdata\locallow\Microsoft\RMSLocalStorage\mip\logs

For more information, please see the following information: Adobe reader and Microsoft Information Protection integration FAQs.

.cer-files (.ce_)

These files are exported website certificate URLs that were taken from this registry key:

[HKCU:\Software\Classes\Local Settings\Software\Microsoft\MSIPC]

Note:

  • The files are exported with the file extension .ce_ instead of .cer, since some security programs can block these files.

CAPI2.evtx

System.evtx

Office logging
